# 经验类 tag 的员工判定(业务情境阅历)

你的任务:对一个员工,通过 `/tag-employee` skill 通读其 profile,按指定 tag_list 逐个判定是否命中,命中的写入 `employee_tag_map`。

本批 tag 都是**经验类**(`kind='experience'`)——指经历过某种业务局面,如`0→1 从无到有搭业务`、`国际化 / 出海`、`商业化变现`、`高速增长期`、`跨部门大型项目`。

## 证据口径:看业务的阶段 / 性质 / 范围,不看技术细节

经验的证据在工作经历正文(`description` / `jobResp` / `companyDesc`)里"**身处并主导 / 深度参与了这种业务局面**"的段落。判定要锚在业务情境本身:

- **业务情境要具体落地**:简历写"在国际化公司",不等于做过国际化业务——要找到"负责某海外市场拓展 / 跨境业务搭建"这种具体经历
- **依据要直接**:`tags.description` 说某种业务情境,profile 里要找到对应的业务阶段 / 性质描述,不能从"在创业公司 → 可能做过 0→1"间接推

## 经验域常见的模糊边界

经验的边界天然容易模糊:`0→1 创业`算独立法人新业务,还是包含成熟公司内部新项目孵化?`商业化变现`算参与过变现链路某一环,还是要主导变现模式设计?短期打下手 vs 主导算不算同一档?

## 完成时输出

```
emp: <emp_id> (<name>)
判定数: 命中 N / 跳过 M
命中: <code>(<一句话 reasoning,引用具体的业务情境段>), ...
跳过(边界模糊): <code>(<一句话:哪条边界没划清>), ...
跳过(其他): <code>(<一句话原因>), ...
```
