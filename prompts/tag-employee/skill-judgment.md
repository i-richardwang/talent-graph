# 技能类 tag 的员工判定(方法论 / 手艺)

你的任务:对一个员工,通过 `/tag-employee` skill 通读其 profile,按指定 tag_list 逐个判定是否命中,命中的写入 `employee_tag_map`。

本批 tag 都是**技能类**(`kind='skill'`)——指做某类技术工作的方法论能力,如`推荐排序`、`因果分析`、`实验设计`、`特征工程`、`模型训练上线`。

## 证据口径:看方法本身的应用经历,不看岗位名 / 团队名

技能的证据在工作经历正文(`description` / `jobResp`)里"**具体用这套方法做了事**"的段落。判定要锚在方法本身:

- **岗位名 / 团队名不算证据**:简历写"算法工程师"或"在推荐团队",不等于本人做过推荐排序——要找到"做了召回/排序/CTR 预估模型"这种具体方法的应用经历
- **依据要直接**:`tags.description` 说某项方法,profile 里要找到对应的实际操作,不能从"在 X 公司 / X 岗 → 可能会 X"间接推

## 技能域常见的模糊边界

技能的边界天然容易模糊:`因果分析`算描述统计 + AB 报表,还是要因果识别 / 反事实建模?`机器学习建模`算调现成库跑一次,还是要独立设计特征 + 选型 + 上线?

## 完成时输出

```
emp: <emp_id> (<name>)
判定数: 命中 N / 跳过 M
命中: <code>(<一句话 reasoning,引用具体的方法应用段>), ...
跳过(边界模糊): <code>(<一句话:哪条边界没划清>), ...
跳过(其他): <code>(<一句话原因>), ...
```
