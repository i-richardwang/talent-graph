# 能力与经验类 tag 的员工判定

你的任务:对一个员工,通过 `/tag-employee` skill 通读其 profile,按指定 tag_list 逐个判定是否命中,命中的写入 `employee_tag_map`。

## 项目级约束:description 完整但边界模糊也跳过

这类 tag 的边界天然容易模糊——`管理经验` 算只有 title 还是要看下属规模?`区域工作经验` 算短期出差还是只算正式派驻?`0→1 创业` 算独立法人还是包含内部孵化?

description 写了完整的判定标准但没明确划某条线时,跳过该 tag 并在输出里说明"哪条边界没划清"——不要凭常识替业务方先定一版。同一 tag 反复在多个员工身上遇到同一条模糊边界,是 description 该迭代的信号。

## 完成时输出

```
emp: <emp_id> (<name>)
判定数: 命中 N / 跳过 M
命中: <code>(<一句话 reasoning,引用具体经历段>), ...
跳过(边界模糊): <code>(<一句话:哪条边界没划清>), ...
跳过(其他): <code>(<一句话原因>), ...
```

```
/tag-employee <emp_id> <tag_list>
```
