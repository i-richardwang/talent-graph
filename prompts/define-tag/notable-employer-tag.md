# 定义一个公司域名单标签

你的任务:对一个公司域名单标签,通过 `/define-tag` skill 完成 tag 字段注册 + 标准公司实体清单挂载。

## 项目级约束:"业内公认" vs "模糊概念"的判定边界

公司域标签里"业内公认 + 边界清晰"和"模糊待消歧"的分线容易跑偏——SKILL 给的是通用标准,这里明确项目级的具体口径:

**业内公认 → 直接做**

数量稀少、有公认 acronym、清单不随时间变——MBB(3) / 四大(4) / 三中一华(4) / BAT(3) / TMD(3) / FAANG / 红圈所 / Magic Circle 这类。用 WebSearch 能稳定核实出同一份清单,不需要消歧。

**含"头部 / 大厂 / 巨头 / 独角兽 / 待补 / 等" 字样 → 视为模糊,必须消歧**

这类标签**任务参数 `[消歧说明]` 必须给定以下任一**,否则 skill 视为清单不成立 → 终止任务等待消歧:

- **具体清单**(如 `互联网大厂 = 字节 / 阿里 / 腾讯 / 美团 / 京东 / 拼多多 / 网易`)
- **权威来源 + 版本**(如 `独角兽 = CB Insights 2024 榜单`)
- **可量化口径**(如 `头部公募 = 截至 2024 Q4 管理规模前 10`)

不靠 Agent 凭常识从多份榜单里挑一份——那是把项目应该决定的口径推给 Agent 猜。

## 完成时输出一行结果

```
tag: <code> (<name>)
状态: created | already_exists
本轮动作: 挂入 N 家 / 解绑 M 家 / 无变更
依据: <一句话来源,如"业内公认清单 (McKinsey/BCG/Bain)" / "任务参数指定清单" / "CB Insights 2024 榜单">
```
