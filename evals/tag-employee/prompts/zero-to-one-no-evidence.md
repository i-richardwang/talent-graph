# 经验类 tag 的员工判定(业务情境阅历)

你的任务:对一个员工,通过 `/tag-employee` skill 通读其 profile,按指定 tag_list 逐个判定是否命中,命中的写入 `employee_tag_map`。

本批 tag 都是**经验类**(`kind='experience'`)——指经历过某种业务局面,如`0→1 从无到有搭业务`、`国际化 / 出海`、`商业化变现`、`高速增长期`、`跨部门大型项目`。

## 证据锚在业务情境本身,不看公司性质 / 行业

经验的证据是工作经历正文(`description` / `jobResp` / `companyDesc`)里"**身处并主导 / 深度参与了这种业务局面**"的段落:`0→1`要找到亲手搭起某条新业务线的经历,`国际化`要找到负责某海外市场拓展 / 跨境业务搭建的经历。

**不算亲历这种业务情境的**:公司性质 / 行业("在创业公司""在出海业务的公司")、只把它当背景提及——这些间接信号不能据以命中,也不能从"在创业公司 → 可能做过 0→1"间接推。

经验的达标线天然容易模糊:`0→1`算独立法人新业务,还是含成熟公司内部新项目孵化?`商业化变现`算参与变现链路某一环,还是要主导变现模式设计?短期打下手 vs 主导算不算同一档?这类"算到什么程度"的线,就是 `borderline` 的归属。

## 完成时输出

```
emp: <emp_id> (<name>)
判定数: 命中 N(confident A / borderline B) / 跳过 M
命中(confident): <code>(<reasoning,引用具体的业务情境段>), ...
命中(borderline): <code>(<reasoning + 卡在哪条边界没划清>), ...
跳过: <code>(<一句话原因:证据不足 / 不属于 / 占位 description>), ...
```
