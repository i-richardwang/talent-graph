# 技能类 tag 的员工判定(方法论 / 手艺)

你的任务:对一个员工,通过 `/tag-employee` skill 通读其 profile,按指定 tag_list 逐个判定是否命中,命中的写入 `employee_tag_map`。

本批 tag 都是**技能类**(`kind='skill'`)——指做某类技术工作的方法论能力,如`推荐排序`、`因果分析`、`实验设计`、`特征工程`、`模型训练上线`。

## 证据锚在方法本身,不看岗位名 / 团队名

技能的证据是工作经历正文(`description` / `jobResp`)里"**具体用这套方法做了事**"的段落:`因果分析`要找到做了因果识别 / 反事实估计的段落,`推荐排序`要找到建了召回 / 排序 / CTR 预估模型的段落。

**不算做了这个方法的**:岗位 / 团队名("算法工程师""在推荐团队")、相邻但不同的事、只把该主题当话题提及(如"做过一份关于推荐的分析"≠"做过推荐排序模型")——这些都不能据以命中,也不能从"在 X 公司 / X 岗 → 可能会 X"间接推。

技能的达标线天然容易模糊:`因果分析`算描述统计 + AB 报表,还是要因果识别 / 反事实建模?`机器学习建模`算调现成库跑一次,还是要独立设计特征 + 选型 + 上线?这类"做到什么程度才算"的线,就是 `borderline` 的归属。

## 完成时输出

```
emp: <emp_id> (<name>)
判定数: 命中 N(confident A / borderline B) / 跳过 M
命中(confident): <code>(<reasoning,引用具体的方法应用段>), ...
命中(borderline): <code>(<reasoning + 卡在哪条边界没划清>), ...
跳过: <code>(<一句话原因:证据不足 / 不属于 / 占位 description>), ...
```
